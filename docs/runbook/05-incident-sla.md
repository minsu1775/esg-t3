# Runbook 05: 인시던트 SLA (메타 Runbook)

> **트리거**: 모든 인시던트 분류·통보·문서화의 표준 절차
> **본문 완성 Phase**: Phase 0 (지금 완성)

## 1. 심각도 분류

| 등급 | 정의 | 인지 SLA | 1차 대응 | 복구 |
|---|---|---|---|---|
| P1 | 데이터 무결성·보안 사고 | 15분 | 1시간 | 4시간 |
| P2 | 핵심 기능 중단 | 30분 | 2시간 | 24시간 |
| P3 | 일부 기능 장애 | 1시간 | 4시간 | 1주 |
| P4 | 경미한 이슈 | 1일 | — | 1주 |

## 2. 커뮤니케이션 템플릿

### 초기 알림 (인지 직후)

```
[P{등급}] {제목} — {YYYY-MM-DD HH:MM KST}
- 영향: {요약}
- 현재 조치: {수행 중인 행동}
- 다음 업데이트: {시각}
- 담당: {이름}
```

### 상태 업데이트 (30분~1시간 간격)

```
[P{등급}] {제목} — UPDATE {YYYY-MM-DD HH:MM}
- 진행: {새로 알게 된 것}
- 다음 단계: {계획}
- ETA: {예상 복구 시각}
```

### 해결 보고 (복구 직후)

```
[P{등급}] {제목} — RESOLVED {YYYY-MM-DD HH:MM}
- 원인: {요약}
- 복구 조치: {수행한 것}
- 포스트모템: {링크} (P1·P2는 24시간 내 작성)
```

## 3. 포스트모템 의무

- P1·P2: 24시간 내 작성 (`docs/postmortems/YYYY-MM-DD-<title>.md`)
- P3·P4: 권장
- 포스트모템 구조: 타임라인 → 원인 분석 → 임팩트 → 액션 아이템
